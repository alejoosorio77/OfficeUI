Exceptions

Whenever your application does not work as desired, you can inspect various exceptions which can be thrown by OfficeUI. In order to see the exception messages, you should open a console window in the browser in which you're currently building your application. When an exception is thrown, you'll find it in the console window which can provide usefull information to help you debugging your application.

When an exception is thrown, you'll always be provided with the type of the exception and a custom message. Based on those 2 values, you should be able to identify the issue which is causing the exception.

Built-in exception types

The table below does contain all the various exceptions that can be thrown.

Type Description
OfficeUIStylesheetException Indicates an error with an OfficeUI stylesheet.
OfficeUILoadingException Indicates an error while loading a file which is needed to render the OfficeUI application.
OfficeUIConfigurationException Indicates a configuration error.

Thrown exceptions

When an exception is being thrown, it's written in the console window of the currently opened browser in a specific format. Each exception is formatted with the following syntax:

Output 
[ExceptionType]: Message

Custom exceptions

In an OfficeUI application, you can also write custom exception messages, which are automatically formatted with the same syntax as above.
When you want write a custom exception message, it can be done with the following code:

Syntax

new OfficeUIException(exceptionType, message);

Parameters

exceptionType
A string defining the type of the exception. Can be any given string.
message
A message describing the error which occured. Can be any given string.

Examples

JavaScript 
var exception = 'CustomOfficeUIException'; // String describing the type of the exception.
var message = 'exception message.'; // Message to pass to the exception.
new OfficeUIException(message); // Throw the exception to the console window.

// Output: [CustomOfficeUIException]: exception message.

// Shorthand call:
new OfficeUIException('CustomOfficeUIException', 'exception message');

// Output: [CustomOfficeUIException]: exception message.
